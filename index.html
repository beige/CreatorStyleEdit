Welcome

This is the project page of CreatorStyleEdit, a plugin for the Qt Creator IDE. It is used to colorize the navigation widgets and the output panes by setting a user defined QPalette on them.

Why not using stylesheets?

I've used QtCreator with the -stylesheet command line argument to get a dark color scheme. Since QtCreator 3.1.1, this doesn't work properly and seems to be a bug in QtCreator (see Qt Bug #12078).

After investigating the QtCreator sources, I've found out that it is using an application wide custom stlye (so called Manhattan Style) and that the Qt documentation states that Qt style sheets are currently not supported for custom QStyle subclasses. So I decided, to write a small plugin which sets the QPalette directly on the widget classes that are not styled by the color scheme for the text editor.

Future Development

Future development could include the possibility to set and edit stylesheets on QtCreator at runtime. This has the advantage, that QtCreator must not be closed and started again for editing the style sheet.

Installation

For installation, just copy the CreatorStyleEdit folder into the plugin directory of QtCreator. Which is located as follows:

Windows

Under Windows, the plugin directory of QtCreator is located in the QtCreator installation under lib\qtcreator\plugins. If you have installed QtCreator along with Qt 5.3.0 in C:\Qt\Qt5.3.0, the plugin directory will be at C:\Qt\Qt5.3.0\Tools\QtCreator\lib\qtcreator\plugins

Linux

If you have installed Qt into your home directory, the path to the plugin directory is similar to the Windows path. As an example, if you have installed Qt into a directory /home/yourname/bin/Qt5.3.0, the QtCreator plugin directory will be at /home/yourname/bin/Qt5.3.0/Tools/QtCreator/lib/qtcreator/plugins

Mac OS X

Under Mac OS X, the directory for QtCreator plugins is located in the Qt Creator.app under Qt Creator.app/Contents/PlugIns. To open the app as directory, navigate to the app, open the context menu and click on Show package content. Now you can see the Contents directory.

Usage

After installation, the dialog of CreatorStyleEdit can be opened with Tools -> Style Editor -> Edit Style. Every line of the dialog modifies another color role of the palette. The rectangle in the center of the line shows the current color of the role and a click on the Select button opens up a color picker dialog.

A click on Apply or Ok sets the current palette of the dialog on all widgets and saves it permanently. On the next startup of Qt Creator, the palette will be loaded from the settings and applied on all widgets.

The button Restore Defaults can be used to reset the palette to the application default palette.

Which Widgets will be styled?

Navigation widgets
  • Projects
  • Open Documents
  • Bookmarks
  • File System
  • Class View
  • Outline
  • Type Hierarchy
  • Include Hierarchy
Output Panes
  • Issues
  • Search Results
  • Application Output
  • Compile Output
  • QML/JS Console
  • General Messages
  • Version Control
Debugging Widgets

All debugging widgets will be styled, which includes e.g. the stack view, locals and expressions and the thread view.
